# Report Output Standards

> **Trigger keywords:** report output, save report, report format, PDF report, Excel report

---

## Output Location

All generated reports save to:
```
~\Projects\your organization\Requested Reports\{YYYY-MM-DD}\
```

### Folder Structure
```
Requested Reports/
├── 2026-01-03/          # Dated folder
│   ├── 2026-01-03_ticketing_tool-companies.csv
│   └── 2026-01-03_crm_tool-report.pdf
├── 2026-01-05/          # Another dated folder
│   └── 2026-01-05_bom_tool-analysis.json
├── .cleanup/            # Staged for deletion (per cleanup rules)
│   ├── temp_scripts/
│   └── CLEANUP_LOG.md
```

---

## Dated Folder Logic (MANDATORY)

1. Get current date: `{YYYY-MM-DD}` format (e.g., `2026-01-06`)
2. Check if folder exists at `Requested Reports\{date}\`
3. If folder **exists**: save report directly into it
4. If folder **does NOT exist**: create it first, then save report

### PowerShell Pattern
```powershell
$date = Get-Date -Format "yyyy-MM-dd"
$folder = "~\Projects\your organization\Requested Reports\$date"
if (-not (Test-Path $folder)) { New-Item -ItemType Directory -Path $folder }
# Save report to $folder
```

## Naming Convention

**Format:** `{YYYY-MM-DD}_{system}_{report-type}.{ext}`

| Example | Description |
|---------|-------------|
| `2026-01-03_ticketing_tool-companies.xlsx` | ticketing-tool duplicate detection |
| `2026-01-03_crm_tool-report-Q4-2025.pdf` | your CRM quarterly sales |
| `2026-01-05_bom_tool-analysis.json` | your catalog service catalog data |

## Format Selection

| Report Type | Format | When to Use |
|-------------|--------|-------------|
| **Status/Performance Reports** | **PDF** | Sales reports, pipeline reports, summaries, dashboards - anything meant to be "read" |
| **Data Extraction/Comparison** | **Excel (.xlsx)** | Duplicate detection, data comparisons, raw exports for further analysis |
| **Ambiguous** | **ASK USER** | If unsure whether PDF or Excel fits better |

**NEVER use .txt files for reports.** Text files are unprofessional and lack formatting.

## PDF Formatting Standards (MANDATORY)

```yaml
Professional PDF Requirements:
  Header:
    - Company/system logo or name
    - Report title
    - Date generated
    - Period covered (if applicable)

  Layout:
    - Clean borders around tables
    - Justified text alignment
    - Consistent typography (1-2 font families max)
    - Proper spacing between sections

  Tables:
    - Alternating row colors for readability
    - Bold headers
    - Right-align numbers, left-align text
    - Currency formatting with $ and commas

  Footer:
    - Page numbers
    - "Generated by Claude Code" attribution

  Responsive:
    - Optimized for both mobile and desktop viewing
    - Readable at standard zoom levels
```

## Rules

- Never output reports to temp folders or tool-result caches
- **Never save .py/.js/code files to Requested Reports** - these are skills, save to `skills/` folder
- Temp scripts: save to system temp, execute, delete immediately
- If a script is reusable, create a skill file in `~/.claude\skills\`
